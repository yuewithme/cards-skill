商业模式画布分析图

把一门生意拆成九个相互咬合的模块：谁买、买什么、怎么触达、靠什么交付、成本和收入是否闭合。

9 Blocks 不是填表游戏，而是检查一套商业假设能不能跑通。
Core Question

这门生意为什么有人愿意付钱，并且企业能持续交付？

商业模式画布的价值，不在于列出九个名词，而在于发现各模块之间是否互相支撑：客户是否清楚，价值是否尖锐，渠道是否到达，成本是否被收入覆盖。

How To Read

先看中间，再看两边，最后看底部。

中间回答价值，右侧回答市场，左侧回答交付，底部回答经济账。任何一个模块写不清，通常都意味着商业假设还没有被验证。

九宫格不是并列清单，而是一张价值流动图

左侧让价值被制造出来；中间定义客户愿意购买的理由；右侧把价值送到客户手上；底部检验钱是否回得来。

关键伙伴

Key Partners
哪些外部角色必须一起完成交付？

供应商、渠道方、技术平台、履约网络、政府或生态合作伙伴。它们决定哪些能力自建，哪些能力外包或联盟。

关键业务

Key Activities
每天必须做对哪些事？

研发、生产、获客、运营、服务、风控、内容、撮合或履约。这里写让价值主张成立的核心动作。

关键资源

Key Resources
什么资产支撑你的壁垒？

品牌、数据、专利、团队、牌照、供应链、现金、用户关系和基础设施。资源越稀缺，模式越难复制。

价值主张

Value Propositions
客户为什么非要选择你？

价值主张要落到具体痛点、收益和替代方案上：是省钱、省时、降低风险、提升效率，还是完成过去做不到的任务。

如果这里只能写成“更好体验”或“一站式服务”，后面的八格都会变成空转。

客户关系

Customer Relationships
你和客户如何持续发生关系？

自助、社群、顾问式服务、会员、长期合同或高频运营。关系形态影响复购、客单价和服务成本。

渠道通路

Channels
客户在哪里知道、购买和使用？

内容平台、销售团队、门店、App、代理商、私域或合作渠道。渠道不是曝光位，而是完整购买路径。

客户细分

Customer Segments
到底服务哪一类人或组织？

按任务、场景、预算、决策链、使用频率和风险承受能力划分。客户越泛，价值主张越容易变空。

成本结构

Cost Structure
为了交付价值，哪些成本不可避免？

固定成本、变动成本、获客成本、研发投入、履约成本、服务成本和渠道分成。成本结构决定规模扩大后利润是变厚还是变薄。

收入来源

Revenue Streams
客户如何付钱，为什么愿意持续付？

一次性销售、订阅、佣金、租赁、广告、增值服务、交易抽成或结果付费。收入方式必须匹配客户价值和交付成本。

左侧：能力系统

关键伙伴、关键业务和关键资源回答“企业凭什么交付”。如果左侧薄弱，右侧承诺越大，履约风险越高。

中间：价值假设

价值主张是整张画布的心脏。它必须明确客户痛点、替代选择和付费理由，否则九格只是漂亮分类。

右侧：市场系统

客户、渠道、关系和收入回答“钱从哪里来”。市场系统越清楚，增长路径和定价逻辑越容易验证。

How To Use

四步使用

01 DEFINE

先写客户细分和价值主张，避免从资源和产品自嗨开始。

02 LINK

把渠道、客户关系和收入方式连起来，检查购买路径是否真实。

03 TEST

逐格标注证据强弱：已验证、待验证、纯假设，不要混在一起。

04 REVISE

优先修正牵动最多模块的假设，例如价值主张、渠道或成本结构。

True Output

真正的输出

商业模式画布最终要产出一个判断：这套模式的增长逻辑、交付能力和经济账是否闭环；如果不闭环，应该先验证哪一个关键假设。

误区：不要把它当作商业计划书目录。画布不是为了把内容填满，而是为了暴露冲突：客户想要的、企业能做的、市场愿意付钱的，是否是同一件事。
商业模式画布不是九个格子的答案，而是一张假设地图：能连成闭环的才是模式，连不上的只是想法。
